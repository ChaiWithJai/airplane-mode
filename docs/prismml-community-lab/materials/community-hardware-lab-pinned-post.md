# Proposed Pinned Post: `#community-hardware-lab`

This channel is for Bonsai runtime and hardware experiments that can become
shared evidence.

Post here if you tested Bonsai on a device, backend, browser, runtime, or
end-user workflow. Please include as much of the benchmark template as you can:

- hardware;
- model artifact;
- backend/runtime;
- command or entrypoint;
- tokens/sec;
- memory or VRAM;
- failure mode;
- what use case you were testing.

The goal is not leaderboard theater. The goal is to learn what Bonsai makes
practical locally, what still fails, and which experiments should become docs,
issues, reference integrations, or end-user case studies.

Especially useful:

- old laptops and low-RAM machines;
- Apple Silicon and iPhone/browser GPU reports;
- CUDA, AMD, ROCm, Strix Halo, and iGPU results;
- ONNX/WebGPU/browser failures;
- llama.cpp, MLX, Rust, and Python runtime comparisons;
- app demos where local inference changes the user workflow.

If your post is incomplete, still post it. Just say what is missing.

Useful framing:

```text
Who can now do what, locally, because this model/runtime/hardware path works?
```
